# Day 7 - Claude Usage Strategy

## Overview
Today, I explored how to use different Claude models effectively by creating my own personalized Claude Usage Strategy. The goal was to understand which model should be used for different types of tasks and how to optimize effort levels for better productivity.

## My Profile
- Name: Anshika Soni
- Final Year B.Tech (Computer Science)
- Learning: Python, SQL, Java, Machine Learning & Data Analytics
- Participating in the ABTalks 60-Day Claude Challenge

## Key Learnings

### Claude Haiku
Best for:
- Quick factual questions
- Definitions
- Formatting text
- Short content writing

### Claude Sonnet 5
Best for:
- Daily coding and debugging
- Machine Learning concepts
- Learning new technologies
- Resume improvements
- Project documentation

### Claude Opus
Best for:
- Complex project planning
- Multi-step reasoning
- Resume & ATS optimization
- Career roadmap creation
- Architecture-level decisions

## Recommended Workflow

| Task | Model | Effort |
|------|-------|--------|
| Coding & Debugging | Sonnet 5 | Standard |
| Machine Learning | Sonnet 5 | Standard–High |
| GitHub Documentation | Sonnet 5 | Standard |
| LinkedIn Posts | Haiku / Sonnet | Low–Standard |
| Resume & ATS | Opus | High–Max |
| Quick Queries | Haiku | Low |
| Project Planning | Opus | High |

## Biggest Takeaways

- Choose the model based on task complexity.
- Don't use Opus for simple work.
- Reserve High/Max effort for important tasks.
- Sonnet 5 is the best default model for everyday work.

## My Final Strategy

**Default Model:** Claude Sonnet 5

**Default Effort:** Standard

This combination offers an excellent balance between reasoning quality, response speed, and efficient usage limits for my daily learning and development.
